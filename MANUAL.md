# Simple RTS AI 게임 설명서

이 문서는 Phase 1부터 Phase 6까지 구현된 게임 기능과 플레이 방법을 정리한 사용자용 설명서입니다.

## 게임 개요

Simple RTS AI는 브라우저에서 실행되는 2D 전술 RTS 프로토타입입니다.
플레이어는 아군 분대를 직접 선택하고 이동시키며, 적을 발견하면 자동 전투를 통해 교전합니다. 또한 Player Mode와 AI Commander Mode 자리표시자 UI를 전환할 수 있습니다.

현재 구현된 핵심 흐름은 다음과 같습니다.

```text
플레이어 마우스 입력
→ Command 객체 생성
→ 게임 명령 처리
→ 유닛 이동 / 공격 / 상태 변경
```

이 구조는 이후 AI Commander Mode에서 자연어 명령을 Command 객체로 바꾸는 방식으로 확장하기 위한 기반입니다.

## 실행 방법

프로젝트 루트에서 간단한 정적 서버를 실행합니다.

```bash
python3 -m http.server 8000
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:8000
```

초기 단계에서는 `index.html`을 직접 열어도 동작할 수 있습니다.

## 기본 화면 구성

- 초록색 격자 배경은 전장입니다.
- 파란색 원형 유닛은 아군입니다.
- 빨간색 원형 유닛은 적군입니다.
- 유닛 위에는 이름이 표시됩니다.
- 유닛 근처에는 체력바가 표시됩니다.
- 선택된 아군은 노란색 선택 원으로 표시됩니다.
- 교전 중인 유닛 사이에는 붉은 공격선이 표시됩니다.
- 상단 모드 패널에서 Player Mode와 AI Commander Mode를 전환할 수 있습니다.

## Phase 1: 기본 전장과 유닛 표시

### 구현된 기능

- HTML5 Canvas 기반 전장 표시
- 2D 상단 시점 전장 배경
- 아군 5개 분대 생성
- 적군 5개 분대 생성
- 유닛을 단순한 원형 심볼로 표시
- 유닛 이름 표시

### 조작 방법

Phase 1의 주된 목적은 화면에 전장과 유닛을 표시하는 것입니다.
이 단계에서는 본격적인 조작보다는 게임이 브라우저에서 정상적으로 열리고 유닛이 보이는지가 중요합니다.

## Phase 2: 단일 유닛 선택과 이동

### 구현된 기능

- 마우스로 아군 유닛 선택
- 선택된 유닛 강조 표시
- 선택된 아군에게 이동 명령 내리기
- 플레이어 입력을 Command 객체로 변환
- Command 처리 시스템을 통해 유닛 이동 적용

### 조작 방법

1. 아군 유닛을 마우스 왼쪽 버튼으로 클릭합니다.
2. 선택된 유닛 주변에 노란색 원이 표시됩니다.
3. 전장의 빈 지점을 마우스 왼쪽 버튼으로 클릭합니다.
4. 선택된 아군 유닛이 해당 지점으로 이동합니다.

### 참고

마우스 클릭이 유닛 위치를 직접 바꾸는 것이 아니라, 이동 Command가 생성되고 게임 시스템이 그 Command를 실행합니다.

## Phase 3: 시야와 적 발견

### 구현된 기능

- 적 유닛은 처음부터 항상 보이지 않음
- 아군 유닛의 시야 범위 안에 들어온 적만 발견됨
- 발견된 적은 화면에 표시됨
- 적의 마지막 알려진 위치 정보 저장 기반 마련
- Fog-of-war 스타일의 기본 은폐/발견 흐름 구현

### 조작 방법

1. 아군 유닛을 선택합니다.
2. 의심되는 지역으로 이동 명령을 내립니다.
3. 아군 시야 범위 안에 적이 들어오면 적 유닛이 화면에 나타납니다.
4. 시야 밖의 적은 보이지 않습니다.

### 플레이 포인트

정찰이 중요해집니다.
적 위치를 알기 위해 아군을 전장 앞으로 이동시켜야 합니다.

## Phase 4: 자동 전투와 기본 적 AI

### 구현된 기능

- 아군과 적군이 서로를 감지하면 자동으로 공격
- 유닛별 체력, 공격력, 공격 사거리 적용
- 체력바 표시
- 교전 중인 유닛 사이에 공격선 표시
- 체력이 0이 된 유닛 제거 또는 비활성화
- 승리 / 패배 상태 표시
- 적 유닛의 기본 AI 행동 추가
- 적은 완전히 가만히 있지 않고 순찰하거나 감지한 아군을 향해 접근

### 조작 방법

1. 아군을 선택하고 적이 있을 것으로 예상되는 방향으로 이동시킵니다.
2. 적이 아군 시야에 들어오면 화면에 나타납니다.
3. 아군과 적군이 공격 가능 조건을 만족하면 자동으로 교전합니다.
4. 모든 적을 제거하면 승리합니다.
5. 모든 아군이 제거되면 패배합니다.

### 전투 규칙

- 유닛이 공격 사거리 안의 적을 찾으면 공격합니다.
- 아군은 보이는 적 또는 전투 판정상 유효한 적을 공격합니다.
- 적도 가까운 아군을 감지하면 접근하거나 공격합니다.
- 공격력, 사거리, 체력은 유닛 기본 스탯에 따라 처리됩니다.

## Phase 5: 다중 선택과 그룹 명령

### 구현된 기능

- 드래그 박스로 여러 아군 유닛 선택
- 여러 선택 유닛 유지
- 선택된 여러 유닛에게 한 번에 이동 명령 내리기
- 그룹 이동 시 목표 지점 주변으로 유닛을 분산 배치
- 선택된 유닛들이 같은 좌표에 겹쳐 서는 현상 완화
- 기본 attackMove 명령 지원
- 우클릭으로 공격 이동 명령 내리기
- 보이는 적을 클릭해 공격 이동 대상으로 지정 가능

### 조작 방법: 여러 유닛 선택

1. 전장에서 마우스 왼쪽 버튼을 누릅니다.
2. 버튼을 누른 상태로 드래그합니다.
3. 화면에 반투명한 선택 박스가 표시됩니다.
4. 선택 박스 안에 들어온 아군 유닛들이 모두 선택됩니다.
5. 선택된 유닛들은 노란색 선택 원으로 표시됩니다.

### 조작 방법: 그룹 이동

1. 여러 아군을 드래그 박스로 선택합니다.
2. 전장의 빈 지점을 마우스 왼쪽 버튼으로 클릭합니다.
3. 선택된 모든 아군이 해당 지점 주변으로 이동합니다.
4. 유닛들은 정확히 같은 좌표로 몰리지 않고, 목표 주변에 간단한 대형처럼 퍼져 도착합니다.

### 조작 방법: attackMove

1. 하나 이상의 아군을 선택합니다.
2. 전장의 목표 방향이나 위치를 마우스 오른쪽 버튼으로 클릭합니다.
3. 선택된 유닛들이 attackMove 명령을 받고 해당 방향으로 이동합니다.
4. 이동 중 적을 발견하면 적을 향해 접근합니다.
5. 공격 사거리 안에 들어오면 멈춰서 교전합니다.

### 조작 방법: 보이는 적 공격 지정

1. 아군을 선택합니다.
2. 시야에 들어와 화면에 보이는 적을 클릭합니다.
3. 선택된 아군은 해당 적을 목표로 공격 이동을 수행합니다.

### 일반 이동과 attackMove 차이

| 구분 | 일반 이동 | attackMove |
|---|---|---|
| 입력 | 빈 지점 왼쪽 클릭 | 오른쪽 클릭 또는 보이는 적 클릭 |
| 목적 | 지정 위치로 이동 | 이동 중 적 발견 시 교전 우선 |
| 적 발견 시 | 전투 시스템에 의해 공격할 수는 있음 | 적 위치를 향해 적극적으로 접근 |
| 사거리 안 진입 시 | 명령상 특별 정지는 없음 | 사거리 안이면 이동을 멈추고 교전 |
| 전투 능력 | 동일 | 동일 |

attackMove는 공격력이나 사거리를 강화하는 명령이 아닙니다.
차이는 전투 능력이 아니라, 이동 중 적을 발견했을 때의 행동 방식입니다.

## Phase 6: 게임 모드 UI

### 구현된 기능

- 게임 상태에 `PLAYER_MODE`와 `AI_COMMAND_MODE` 모드 추가
- 상단 UI에 Player Mode / AI Commander Mode 전환 버튼 추가
- 현재 활성 모드를 텍스트와 활성 버튼 스타일로 표시
- AI Commander Mode 선택 시 향후 자연어 명령 입력을 위한 placeholder 패널 표시
- AI Commander placeholder 입력 영역은 아직 명령을 실행하지 않음
- LLM API 호출, API 키 저장, 백엔드 연동은 추가하지 않음
- Player Mode에서는 기존 마우스 선택, 이동, attackMove 조작 유지
- AI Commander Mode에서는 직접 마우스 전장 조작을 비활성화해 향후 Commander 명령 흐름과 구분

### 조작 방법: 모드 전환

1. 화면 상단의 `Player Mode` 버튼을 누르면 직접 조작 모드가 활성화됩니다.
2. Player Mode에서는 기존처럼 아군을 클릭 또는 드래그로 선택하고, 왼쪽 클릭 이동 / 오른쪽 클릭 attackMove를 사용할 수 있습니다.
3. `AI Commander Mode` 버튼을 누르면 AI Commander placeholder 패널이 표시됩니다.
4. placeholder 입력 영역에는 향후 전술 목표를 입력할 예정이지만, 현재 Phase 6에서는 입력 내용이 파싱되거나 Command로 실행되지 않습니다.
5. 다시 `Player Mode` 버튼을 누르면 기존 직접 조작을 계속 사용할 수 있습니다.

### 현재 AI Commander Mode의 한계

Phase 6의 AI Commander Mode는 UI와 상태 전환 기반만 제공합니다.
실제 자연어 파싱, 규칙 기반 명령 해석, LLM 호출, AI 자동 지휘 행동은 아직 구현되지 않았습니다.


## 현재 승리 목표

- 아군 유닛을 조작해 적 유닛을 찾아냅니다.
- 발견한 적과 교전합니다.
- 모든 적 유닛을 제거하면 승리합니다.
- 모든 아군 유닛이 제거되면 패배합니다.

## 현재 구현 범위 밖의 기능

아래 기능들은 아직 구현되지 않았거나 이후 Phase의 목표입니다.

- 자원 채집
- 건물 건설
- 유닛 생산
- 업그레이드
- 복잡한 경로 탐색
- 실제 AI Commander 명령 실행
- 자연어 명령 파싱
- LLM 기반 명령 해석
- 백엔드 API 연동

## 플레이 팁

- 처음에는 적이 보이지 않으므로 아군을 이동시켜 정찰해야 합니다.
- 여러 유닛을 한 번에 움직이려면 드래그 박스 선택을 사용하세요.
- 단순히 이동만 하고 싶으면 왼쪽 클릭 이동을 사용하세요.
- 전진하면서 적을 만나면 싸우게 하고 싶으면 오른쪽 클릭 attackMove를 사용하세요.
- 적이 보이면 그룹을 선택한 뒤 적을 클릭하거나 attackMove로 접근시키세요.
- AI Commander Mode는 현재 placeholder이므로 실제 유닛 조작은 Player Mode로 돌아와 수행하세요.
- 유닛들이 너무 겹치지 않도록 그룹 이동 시 목표 주변에 자동으로 흩어집니다.
